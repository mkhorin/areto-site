View

It is much easier to generate HTML response in separate templates, than directly in controller methods.

Adding a template

Add a section of the application components in the configuration file and specify a view engine.

config/default.js


        

          Create views directory in the project root.
          There put template files containing HTML page layouts.
        

        

          Each controller has its own subdirectory in the views that contains templates of action.
          For example, DefaultController controller and actionNewPost action
          has view/template/default/new-post.ejs template.
        

        
view/template/default/index.ejs

        

        
The first line includes parent page layout.

        
Layout

        

          Layout is a template with common elements that form the structure of the page, frame content.
          Typically, layouts are kept in a separate template file that is located in view/template/_layouts/.
          A layout can be included to a top-level layout.
        

        
<%- body -%> - place the template content will be located (or nested layout).


        
view/template/_layouts/content.ejs

        


        
view/template/_layouts/main.ejs

        


        
Template render

        

          In actionIndex method of the DefaultController
          replace the output message to render the appropriate template.
          By default, the template is searched in view/template/default/ directory controller.
