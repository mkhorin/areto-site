Admin user model

In the admin module you can update a user created by a registration form and to create new ones. A user class inherits from User of app frontend.

User password is required to create a model and is optional for editing. It provides validation scenarios assigned to rules when creating {on: ['create']}. Thus, password attribute is mandatory only with ['create'].includes(model.scenario) condition. If scenario for a rule is not specified, the rule will be applied in all cases.

The Range validator used for status, role attributes to check a value belonging to a set.
