Доступ к модулю администрирования

Для защиты модуля администрирования от неавторизованного доступа, в объектах класса Admin применяется фильтр areto/filters/AccessControl. Фильтр базируется на правилах, определяемых в параметре rules. Каждый элемент правила инициализирует объект класса areto/filters/AccessRule.

Свойство allow определяет тип правила - разрешающее (true) или запрещающее (false). Свойство permissions содержит массив элементов прав доступа (разрешений и ролей), для которых выполняется правило. Помимо элементов доступа определенных в rbac/items, доступны две встроенные роли:

  • ? - обозначает гостевого (анонимного) пользователя.
  • @ - обозначает залогиненного пользователя.
