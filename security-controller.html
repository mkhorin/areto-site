Security controller

A blog user has following actions: sign up, sign in or sign out. All of them are united in the security controller.

controller/AuthController.js


        
Captcha

        

          In addition to internal methods, controller actions can be defined as a separate class.
          It uses a constant ACTIONS. Name of captcha property determines a route.
          Property value determines parameters to create a class instance.
          In this case, a class action is a captcha to protect the blog from automatic registration.
        

        
controller/AuthController.js

        

        
Sign up

        

          Registration and authentication is available for anonymous (isGuest) users only.
          Logged user will get exit link.
        

        

          The SignUpForm form model is instantiated to sign up.
          If the current request already contains form data (isPost), the model gets these values
          and calls registration method. Otherwise, a blank form is displayed.
        

        

          If errors occur the current form is displayed with a warning message.
          If successful, a guest user is redirected to the login page.

Authentication

        

          To access the blog functionality visitors use the authentication form.
          After successful validation, users are forwarded to initiated page.
